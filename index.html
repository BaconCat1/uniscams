What is this list? Why did you make it?
This is a unified list of scammers listed on semi-anarchy servers. Because the original lists are static, and users change their usernames over time, it can be unclear if a user is a scammer or not based on the original lists alone. This list takes the scammers collected in the original lists, and stores them as UUIDs first in order to show who is a scammer right now in an automatic and up to date fashion.
Additionally, the goal was to unifiy the various scammer lists, as one goal of marking scammers is to be punitive as well as a safety measure. Avoiding a scammer from another server is often preferred.
I made this list to show their current LIVE username, as well as the username history, as the static lists maintained on trading Discords suffer from username changes and deliberate obfuscation by scammers. In short, if you're looking to trade with someone, and the name they have right now doesn't show up in the list or username history, you can feel confident they're not a known scammer.
Where are the original scammers lists?
Each server has it's own separately run trading Discord with it's own unique list of scammers. I manually went through and added the data from those servers to this list, sometimes using chat bridge history and/or the ForestBot database to determine what UUID the scammer has. You can check my work by joining the relevant Discord linked below:
  • Refined Vanilla
  • Simply Vanilla
  • Uneasy Vanilla
  • Purity Vanilla
  • Anarchy Network

Once you've joined, you can click the server icon next to a username to be taken directly to the scammer list of that server's trading Discord.
How do you know someone actually scammed though?
This list isn't meant to adjudicate scammer status. The goal is to record scammer's UUIDs, which never change, and present their current username in an easily text searchable way, as well as their username history. If you believe you don't deserve to be marked as a scammer, you'll have to resolve that with the relevant trading Discord mods as well as the person you scammed.
How can you be certain of a scammer's UUID?
Sites like NameMC allow you to see a username's history across any UUIDs it's been used with. Naturally, if a username listed on a scammer list only has one entry, you can be confident that UUID is the scammer. Things get more difficult when multiple UUIDs have used the same username. If the list is a decent record, you can use the time the scammer was listed to narrow when the username was in use, and that often is enough to determine the correct UUID. I also leveraged the data in the ForestBot database since UUIDs are recorded for things like deaths and advancements, which could give an objective link between a username and UUID, as well as a timestamp.
Unfortunately, several lists were not as granular as the Simply Vanilla list, Purity's in particular was quite annoying. I think there's some history behind that, some controversy about what trading Discord was "official", as well as the chat bridge not being as historical as Simply's. I did my best, but where things were unclear I had no choice but to not list them. Better a scammer goes undetected, for now, than an innocent person be marked in my opinion.
You say you're confident about the UUIDs, and yet you list some as just usernames without UUIDs. Why is that?
That's true. The Purity scammer list in particular was not helpful when it came to linking the scammer's username with a UUID. Without additional context and multiple UUIDs having used the username during the server's time online, I couldn't narrow it down. The next best thing was to list the known names since that much is concrete.
NameMC allows people to hide username history, how did you find their UUID anyway?
NameMC isn't the only site with username history. I'd say it's probably the most comprehensive and easiest to use, but being the most popular does lead to users of all stripes trying to hide their username history. Between other username lookup sites, the chat bridges, and the ForestBot database, you usually can't hide from me, unless the original list sucks in the way I described previously.
Why is the username history incomplete for some users?
The API that pulls username data doesn't have a perfect record of usernames. Mojang cut off access to the official API and not all username sites have the same data. One thing you can count on is their current username being the one displayed though.
Is Discord account information being pulled live like Minecraft UUIDs?
No. I couldn't find a clean way to pull live Discord account data without a bot sitting in the middle, and the goal was for this page to be static so that was a no go. The compromise is the bot reads the Discord account info based on the data.json file the page uses, and generates a discord.json file with the Discord account information. While it isn't pulling it live, Discord User IDs are static in the same way Minecraft UUIDs are, which means opening the account from this page in discord by clicking on it will get you the correct account, even if the scammer has changed their Discord username/avatar.
Did you use any other sources beyond the scammer lists?
Yes. You can view those here. Notably, the alts listed in the "UneasyVanilla Coordinate Leak" were used where applicable, as I've found the data listed there valid. However I would caution against loading the site to avoid potential IP Loggers etc.
Why does the page take so long to load? Why do some users fail to load?
The initial load pulls a lot of data from the API I'm using to resolve usernames and username history. Sometimes the API refuses to send data to you because so much is being requested at once. If retries fail, you're simply prompted to attempt to reload them and that's usually good enough. After that, most of the data is cached.
