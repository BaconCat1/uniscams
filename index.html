Before you trade

How to use this list responsibly

01

Search every identity

Check the Minecraft name, any older names you know, UUIDs, alts, and Discord accounts. Scammers can change names; UUIDs do not change.

02

Open the details

Use username history, alt sections, Discord accounts, and server badges to understand why a person appears here.

03

Resolve disputes at the source

This site does not decide whether someone scammed. Handle removals, appeals, and new reports through the relevant trading Discord moderators.

Common warning signs

  • They recently changed names or ask you to ignore an older alias.
  • Their Discord identity does not match the Minecraft account you expected.
  • They push for an off-channel trade or refuse the server's usual process.
  • They use an account that appears as an alt or ambiguous entry.

If a match appears

  1. Do not continue the trade until you understand the entry.
  2. Open the source server badge when available and review the original context.
  3. Ask the relevant trading Discord moderators how they want reports or appeals handled.
  4. Save names, UUIDs, Discord IDs, screenshots, and timestamps before leaving the conversation.
What is this list?
This is a unified list of scammers listed on semi-anarchy servers. The original lists are static, while players can change usernames. This page stores confirmed Minecraft UUIDs where possible and resolves current usernames so a trader can search the person in front of them right now.
Where are the original scammer lists?
Each server has its own trading Discord and its own scammer list. Join the relevant Discord to check the source material or report through that community's process.

How do you know someone actually scammed?
This list is not meant to adjudicate scammer status. It records entries from existing server scammer lists and presents current usernames, username history, alts, and Discord identifiers in one searchable place. If you believe an entry is wrong, resolve it with the relevant trading Discord moderators.
How can a UUID be confirmed?
Name history tools such as NameMC, chat bridge history, and ForestBot data can connect a listed username to a Minecraft UUID. When multiple UUIDs used the same name and the source was not clear enough, this site may leave an entry unlinked instead of attaching the wrong person.
Why are some entries usernames without UUIDs?
Some source lists do not include enough context to connect a name to one specific UUID. Those entries are shown as unlinked or ambiguous so the warning remains visible without pretending the identity is certain.
Why can username history be incomplete?
Mojang no longer exposes the same public username history API, and third-party records differ. Current username resolution is the most important signal, but older names may be missing from some profiles.
Is Discord account information live?
The page attempts to resolve Discord User IDs through a live API. If that fails, it uses the bundled discord.json fallback generated from the page data. Discord profile links still point to the current account for that ID.
Why can loading take time?
The first load fetches live Minecraft and Discord data. Results are cached in your browser for a few hours, so repeat visits should be faster unless you clear cached data.
